AIOS Crash Course

Claude features, ranked

I run my whole operation through Claude Code. So I ranked every feature by one test: how much it changes my day. The scale runs from D (skip it) up to S (can’t work without it). The twelve up top are my S tier, the ones I open daily. Everything else ranks below them.
The daily drivers — my S tier
1
Skills
The #1 leverage unlock. A skill is a reusable recipe (a markdown file) for anything you do more than once. Build your own, pull others’, chain them together. Works in Code, Chat, and Co-work.
▸

HowDrop a markdown file in .claude/skills/ (or grab someone else’s), then call it with /skill-name — or let Claude trigger it automatically when it’s relevant.

WhyAnything you find yourself explaining to Claude more than once becomes a one-line command instead.

AwesomeIt’s just markdown — easy to write, share with your team, and chain skills into each other.

2
Status line /statusline
The most slept-on feature. Always-on view of your model, effort level, and how much of the context window you’ve used — so you know exactly when to clear, compact, or hand off.
▸

HowRun /statusline and describe what you want shown — or run it bare to auto-configure from your shell prompt.

WhyYou always know your model, effort level, and how full your context window is, at a glance.

AwesomeOne tiny line tells you exactly when to clear, compact, or hand off — no more guessing.

3
Routines /schedule
Schedule an actual agent on a recurring basis — not just a deterministic script. Set it once and it runs on its own.
▸

HowRun /schedule and describe the task and how often — Claude walks you through the setup.

WhyRecurring work (daily reviews, status checks) runs without you having to remember or be at your machine.

AwesomeYou’re scheduling a real agent that thinks each run — not a dumb script.

Heads upruns on Anthropic’s cloud — that’s what separates it from the session-bound /loop.

4
Remote control /remote-control
Keep your local Claude Code session going from your phone or the web. Step away, keep building.
▸

HowRun /remote-control, then open the link or scan the QR code on your phone or at claude.ai/code.

WhyContinue a session on the go without moving the work off your machine.

AwesomeThe work stays local — all your tools and project config come with it.

Heads upyour computer has to stay running, and it can’t be used at the same time as Ultraplan.

5
Loop /loop
Repeat a prompt on an interval (it sets up a cron under the hood). Session-bound; auto-expires after about seven days.
▸

How/loop 5m check the build repeats every five minutes; drop the interval and Claude paces itself.

WhyRe-run a check, build, or watch task on a timer while the session stays open.

AwesomeSet it once and forget it — it cleans up after itself in about a week.

Heads uptied to that terminal — close it and the loop stops.

6
Sub-agents /agents
Parallel workers that report back to your main session. Defined as markdown files, invoked like skills. Great for fanning out independent work.
▸

HowRun /agents to manage them, or define one as a markdown file in .claude/agents/ — then Claude fans tasks out to them.

WhyRun focused side tasks (testing, review, research) in parallel without cluttering your main chat.

AwesomeEach gets its own context window and hands a clean summary back.

7
Rewind /rewind
Roll code and the conversation back to an earlier checkpoint. When Claude gets something wrong, rewinding beats correcting it inline — cleaner context, fewer tokens.
▸

HowRun /rewind, then pick the earlier point in the timeline you want to jump back to.

WhyUndo a wrong turn instead of arguing with Claude to fix it.

AwesomeIt rolls back your code and the conversation together, in lockstep.

8
Agent teams
Personas that debate each other to consensus — panels, devil’s advocates, brainstorms. Unlike sub-agents, teammates talk to each other and share a task list.
▸

HowTurn on the experimental flag, then ask Claude in plain English to “create a team” of personas to explore or debate something.

WhyHear several perspectives argue to a conclusion — a built-in panel for a tough call.

AwesomeThe teammates message each other and coordinate, not just report back.

Heads upexperimental — set CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 first, and expect heavy token use.

9
Auto-memory /memory
Claude periodically reviews past sessions and updates its own memory without being asked — it gets smarter on its own.
▸

HowRun /memory and switch auto-memory on.

WhyClaude learns your corrections and patterns over time without you hand-editing CLAUDE.md.

AwesomeIt quietly gets sharper between sessions — no prompting required.

Heads upstored per-repo in .git/claude-memory.json — don’t commit it.

10
Insights /insights
A 30-day report on how you use Claude Code — what’s working, what’s slowing you down, quick wins, and skills worth building.
▸

HowRun /insights in any session.

WhySee how you’ve actually been working over the last 30 days, not how you think you have.

AwesomeIt points out quick wins and skills worth building — fuel for your next automation.

Heads uppaid plans also get a cost and per-tool breakdown.

11
Ultra plan /ultraplan
Offload planning for a big task to the cloud with multiple planning agents — and keep working locally while it thinks.
▸

HowRun /ultraplan with your big task, review the plan in the browser, then execute it.

WhyGet a serious, multi-step plan for a large job without burning your local token budget.

AwesomeCloud agents do the planning while you keep working in the terminal.

Heads upcan’t run at the same time as Remote control.

12
Goal /goal
Run until a clear definition-of-done is met, with verification built in. Set objective criteria, then walk away — it doesn’t have to run for hours to be useful.
▸

How/goal "tests pass and no type errors" — Claude works and verifies until that’s true.

WhySet a clear definition of done and let it run unattended.

AwesomeIt checks its own work, so you can walk away and trust the result.

What to actually take from this
  • Rankings are personal. Sort by how much a feature changes your day-to-day, not by what’s popular.
  • You don’t have to write software to live in Claude Code. It works just as well as an executive assistant and an operating system for knowledge work.
  • Skills are where the compounding is. Every prompt you repeat is a skill waiting to be written.
  • Get token-aware. The status line, /context, rewinding, and prompt caching all come back to spending your context window deliberately.
